# ZeroSort

**AI驱动的桌面笔记应用** — 本地优先存储、富文本编辑，以及可选的端到端加密云同步。

ZeroSort 在 macOS、Windows 和 Linux 上原生运行。笔记存储在本地计算机的 SQLite 数据库中；可选择同步到任何兼容 S3 的存储系统，且所有数据在离开设备前均会进行加密。

网站: [zerosort.app](https://zerosort.app)

## 核心功能

### 智能笔记组织

编写笔记，剩下的交给 AI 处理。ZeroSort 可使用您选择的 AI 服务商自动生成标题、摘要并将其分配至特定文件夹中。无需手动操作，笔记即可被分类至层级目录结构中。

### 富文本编辑

由 Tiptap 3 驱动的全功能编辑器，支持 Markdown、表格、任务列表、数学公式、语法高亮代码块、嵌入图表、思维导图、Mermaid图表以及自由手绘白板。

### 双向链接笔记

使用 `[[Note Title]]` 创建 wiki 风格链接，或使用 `[[Note Title|Display Text]]` 为其设置别名。ZeroSort 会根据现有笔记解析链接，在您输入时提供自动补全，允许您在内联链接之间跳转，并在专门的“链接”面板中同时显示出链（Outgoing links）和反向链接（Backlinks）。失效的链接会单独突出显示，以便您发现对尚未创建的笔记的引用。

### 灵活的标签系统

使用可复用且支持颜色编码的标签组织笔记，标签可与文件夹协同工作。您可以为笔记分配多个标签，在编辑时内联创建新标签，通过专门的标签管理器管理您的标签库，并使用“匹配任意”或“匹配所有”过滤器来缩小大型合集的检索范围，而无需更改文件夹结构。

### 内置规划工具

在笔记中直接添加结构化的规划区块。ZeroSort 内置包含日、周、月、年和列表视图的事件日历，以及带有可拖拽列和卡片的看板。这两个区块均可就地编辑，并包含用于结构化修改的 AI 辅助工作流。

### 加密云同步

使用任何兼容 S3 的存储服务，跨设备同步笔记、标签和受管理的上传图片。所有同步数据在离开您的机器前，均使用 AES-256-GCM 进行端到端加密。三方比较（Three-way comparison）算法可确保安全且能感知冲突的合并。

### 多服务商 AI 支持

连接到 10 种类型的 AI 服务商——OpenAI、Anthropic、Google Gemini、DeepSeek、Mistral、MoonshotAI、Ollama（本地）、OpenRouter、兼容 Grok 的端点，以及任何兼容 OpenAI 的端点。您可以自由切换模型，或使用 Ollama 完全离线运行。

### 自动笔记组织

当您创建笔记时，AI 会处理内容并生成：

- **标题**：简洁的标题（英文最多 6 个词，中文最多 10 个字符）
- **摘要**：对笔记内容的 2-3 句话概述
- **文件夹路径**：层级式的文件夹存放位置（如 `工作 > 会议 > 纪要`）

AI 会考虑您现有的文件夹结构以进行一致的分类。所有生成过程均实时发生，并带有流式预览。

### 两种 AI 模式

ZeroSort 在编辑器内包含两种不同的 AI 工作流，您可以借此转换选定的文本，或与模型进行自由对话。

### 划词模式

在编辑器中选择任何文本即可打开 AI 浮动菜单，对高亮段落执行具有上下文感知的操作。在重写选定内容时，您可以选择使用当前笔记的其余部分作为补充上下文。

| 操作 | 描述 |
| --- | --- |
| **优化** | 提升写作质量和清晰度 |
| **校对** | 修正语法、拼写和标点 |
| **解释** | 对所选文本提供清晰的解释 |
| **翻译** | 翻译为英语、中文、西班牙语、法语、德语或日语 |
| **改变语气** | 以专业、休闲、友好、自信或正式语气重写 |
| **扩写** | 增加更多细节和示例以扩展内容 |
| **缩写** | 在保留关键信息的同时精简文本 |
| **简化** | 使用更简单的语言重写 |
| **生成思维导图** | 将文本转换为可视化的思维导图 |
| **生成Mermaid图表** | 创建 Mermaid 图表（流程图、时序图、类图等） |
| **生成图表** | 创建数据可视化（折线图、柱状图、饼图、雷达图、气泡图） |
| **自定义提示词** | 对所选文本应用任何自定义指令 |

### 提问模式

当您想要进行自由对话而不是转换选定的段落时，请使用编辑器中的浮动“提问 AI”按钮。

- 询问关于正在编写的笔记的开放式问题
- 输入 `@` 引用其他打开的笔记作为补充上下文
- 先生成回答，然后将其插入笔记的任意位置

提问模式非常适合用于头脑风暴、起草新章节、总结多个打开的笔记，或生成与特定选段无关的内容。

### 针对嵌入区块的 AI 助手

ZeroSort 还为嵌入的看板和事件日历配备了专属的 AI 助手。这些助手可以提出结构化的更改建议，对其进行预览，并在笔记内部直接将更新应用于看板项目或日历事件。

## 云同步

### 三方比较算法

ZeroSort 的同步引擎使用三方比较模型进行安全、确定性的同步：

```
本地状态 (SQLite) ──┐
                    ├── 集合 ── 计划 ── 执行
远程状态 (S3) ──────┤
                    │
先前的同步记录 ─────┘
```

**阶段：**

1. **收集** -- 从本地数据库、S3 存储桶和先前的同步记录中获取当前状态
2. **集合** -- 将所有三种状态合并为统一的比较实体
3. **计划** -- 使用 41+ 个决策分支确定每个实体的正确操作
4. **防护检查** -- 在执行前根据安全规则验证计划
5. **执行** -- 按照可配置的并发量执行上传、下载和删除

### 冲突解决

| 策略 | 行为 |
| --- | --- |
| **保留最新** (默认) | 自动保留具有最新修改时间的版本 |
| **保留本地** | 始终优先保留本地更改而非远程更改 |
| **保留远程** | 始终优先接受远程更改而非本地更改 |

### 同步方向

| 模式 | 行为 |
| --- | --- |
| **双向同步** (默认) | 本地和远程之间进行完整的双向同步 |
| **仅推送** | 仅上传本地更改，不下载远程更改 |
| **仅拉取** | 仅下载远程更改，不上传本地更改 |

### 安全防护

- **删除阈值**：如果超过 30% 的项目将被删除，则阻止同步
- **大量删除警告**：当有 10 个以上项目被标记为删除时发出警告
- **空状态保护**：防止意外擦除所有数据
- **同步预览**：在执行前审查所有计划的更改

### 兼容 S3 的存储

可与任何兼容 S3 的存储服务商配合使用：

- Amazon S3
- MinIO
- Backblaze B2
- DigitalOcean Spaces
- 任何兼容 S3 的端点

## 安全与加密

### 端到端同步加密

所有数据在离开您的设备之前均会被加密：

| 参数 | 值 |
| --- | --- |
| **算法** | AES-256-GCM（认证加密） |
| **密钥派生** | 100,000 次迭代的 PBKDF2 算法，配合 SHA-256 |
| **盐值** | 16 字节，每次加密操作随机生成 |
| **随机数 (Nonce)** | 12 字节，每次加密操作随机生成 |

**加密字段**：笔记标题、摘要、内容和目录名称。
**明文字段**：ID、时间戳和结构引用（同步逻辑所必需）。

### 凭据存储

API 密钥和 S3 凭据使用以下方式存储在本地：

| 参数 | 值 |
| --- | --- |
| **算法** | AES-256-GCM |
| **密钥派生** | 机器 ID 与应用盐值的 SHA-256 哈希 |
| **存储格式** | 二进制文件（bincode序列化） |

凭据绑定至本机——它们无法在不同设备上被解密。

### 隐私架构

- **本地优先**：所有数据存储在您设备上的 SQLite 中
- **无遥测**：没有任何使用数据会离开应用
- **可选同步**：云同步完全由用户自愿开启
- **客户端 AI**：AI 的 API 调用直接从您的设备发送给服务商——无中间服务器

## 文件夹与笔记管理

### 层级文件夹

- 无限的嵌套深度用于文件夹组织
- 创建、重命名、移动和删除文件夹
- 具备展开/折叠和虚拟化渲染的文件夹树
- 可选显示每个文件夹的笔记数量
- AI 建议的文件夹放置位置会尊重您现有的结构
- 具有循环依赖和冲突检测的移动验证
- 文件夹删除选项：将笔记移动至“未分类”或永久删除

### 笔记操作

- 手动或通过 AI 生成创建笔记
- 独立编辑标题、摘要、内容、文件夹归属和标签
- 支持事后修改创建日期
- 在笔记之间创建 wiki 风格的链接，支持自动补全和别名
- 在文件夹之间移动笔记
- 直接在笔记头部添加或移除标签
- 支持同时处理多篇笔记的多标签页界面
- 用于批量选择笔记和批量操作的多选模式
- 跨会话保持标签页状态
- 带有确认对话框的未保存更改检测
- 单篇笔记滚动位置持久化
- 在笔记视图中查看出链、分组反向链接以及失效链接
- 支持对 AI 管理的笔记字段进行批量重新生成

### 搜索与导航

- **侧边栏搜索**：跨文件夹名、笔记标题和摘要进行实时搜索
- **标签过滤器**：按一个或多个标签过滤笔记，无需离开当前视图
- **匹配模式**：在“匹配任意”（OR）和“匹配所有”（AND）标签过滤模式之间切换
- **排序选项**：按创建日期或最后修改日期排序
- **日期选择器**：按特定的日历日期过滤笔记
- **链接笔记导航**：从内联 wiki 链接、出链或反向链接直接跳转至被引用的笔记
- **时间轴滑块**：可视化的逐月时间轴，用于在大型笔记库中快速导航浏览

## 标签与过滤

ZeroSort 包含一个专用的标签系统，旨在补充文件夹而非取代它们。

### 标签管理

- 在专用的标签管理器中创建、重命名、重新着色、搜索和删除标签
- 应用可选颜色，使标签组在视觉上更易于快速浏览
- 查看每个标签的笔记数量，以了解标签的使用情况
- 在重组大型标签库时批量选择和批量删除标签
- 安全删除功能会自动从所有受影响的笔记中移除已删除的标签

### 笔记标签工作流

- 为单篇笔记分配多个标签
- 在编辑笔记时内联创建新标签
- 直接从笔记中移除标签，无需打开单独的设置页面
- 使用 AI 为现有笔记重新生成推荐的标签
- 在笔记卡片和打开笔记的工作区视图中显示标签，以便快速扫描

### 过滤与发现

- 从侧边栏或标签管理器激活标签过滤器
- 保持多个标签同时处于活动状态以进行跨维度的视图查看，如 `Research` + `Draft`
- 根据您需要获得更广泛还是更精确的结果，在 OR（或）和 AND（与）匹配之间进行切换
- 一键清除所有活动的过滤器

### AI 与可移植性

- 使用 AI 为现有笔记重新生成推荐的标签
- AI 生成的标签名称可以解析为现有标签，或在需要时自动创建新标签
- 通过 Markdown frontmatter（前言）导入和导出标签，以实现便携式备份和迁移

- 可配置的提交快捷键（Enter、Shift+Enter、Ctrl+Enter）
- 在列表视图中显示/隐藏笔记摘要
- 显示/隐藏文件夹笔记计数
- 在编辑器中显示/隐藏字符和单词统计

- 为自托管端点自定义基础 URL
- 全局开启/关闭 AI 功能
- 网络连接测试以验证服务商连通性

### 数据管理

- **导入**：导入单个 Markdown 文件或完整的 Markdown 文件夹层级

- **清理工具**：预览并从库中移除未使用的标签或空文件夹

## 前置要求

- **Node.js** 20+（推荐 LTS 版本）
- **pnpm** 10.17.1（固定在 `package.json` 中）
- **Rust** 稳定版工具链（1.77+）

## 快速开始

```bash
pnpm install
pnpm tauri:dev    # 完整的桌面应用 (Vite + Rust宿主)
# 或者
pnpm dev          # 仅运行Vite渲染端 (http://localhost:3000)
```

## 脚本命令

| 命令 | 描述 |
| --- | --- |
| `pnpm dev` | 启动 Vite 开发服务器（端口 3000） |
| `pnpm tauri:dev` | 在开发模式下运行 Tauri 桌面应用 |
| `pnpm build` | 生产环境渲染端构建 + 类型检查 |
| `pnpm tauri:build` / `pnpm build:tauri` | 构建并打包生产环境的 Tauri 应用 |
| `pnpm test` | 在监听模式下运行 Vitest |
| `pnpm vitest run` | 运行一次所有测试 |
| `pnpm format` | 使用 Prettier 进行格式化 |

## 项目结构

```
src/                 # React渲染层
  components/        # 按业务域划分的UI组件
  lib/               # 业务逻辑（AI、同步、主题等）
  lib/desktop-adapter.ts  # 渲染层 ↔ Tauri宿主通信桥梁
  locales/           # i18n国际化翻译JSON文件
  routes/            # TanStack Router页面
  store/             # Zustand状态切片
  types/             # 共享的TypeScript类型
src-tauri/           # Tauri v2 Rust宿主端（SQLite、同步、凭据、图片处理）
```
